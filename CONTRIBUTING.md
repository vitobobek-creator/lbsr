# Contributing to lbsr

Contributions are welcome through a public issue before a pull request.

Every contribution must preserve four safeguards:

1. no synthetic record may be described as a recovered respondent;
2. evidence provenance may not be silently discarded;
3. conflicts and validation failures may not be overwritten by user overrides;
4. new functions require tests and user-facing documentation.

Use semantic commit messages, run `R CMD check`, and include a short note on
methodological implications when changing schemas or validation logic.
